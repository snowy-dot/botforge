❓ Frequently Asked Questions

Click any question to see the answer:

🤔 What if my bot doesn't come online?

Try these fixes:

  1. Check that your token is correct (no extra spaces)
  2. Enable "Message Content Intent" in Discord Developer Portal
  3. Make sure your code uses process.env.DISCORD_TOKEN
  4. Check the logs above for error messages
  5. Verify your bot was invited to a server
💸 Is this really free?

Yes, 100% free forever! We use free hosting tiers from Netlify (website) and Render (backend). There are no hidden costs, no premium plans, and no credit card required.

🔒 Is my token safe?

Yes! Your token is:

  • Never shared with anyone
  • Only used to run YOUR bot
  • Deleted after 1 hour
  • Never logged or stored permanently

⏱️ Why is the first start slow?

The first time you start a bot, it needs to install discord.js (the library that connects to Discord). This takes 5-10 seconds. After that, subsequent bots start in 3-5 seconds!

🚫 Can I get banned for using this?

No! You're using your own bot with your own token. This is the same as running the bot on your own computer, just on our servers. Discord doesn't care where your bot runs, as long as it follows their Terms of Service.

🛠️ What if my bot crashes?

Check the Live Logs section for error messages. Common issues:

  • Syntax errors in your code
  • Missing dependencies in your code
  • Wrong intents enabled
  • Invalid token permissions
Fix the issue and click "Start Bot" again!
📊 How many bots can I run?

On the free tier, you can run 1 bot at a time. Stop your current bot before starting a new one. We're working on supporting multiple bots in the future!

✨ Best Practices & Pro Tips

🎯 Before You Start:
  • Test your bot locally first (on your PC)
  • Use environment variables for sensitive data
  • Add error handling to your code
  • Enable required intents in Discord Developer Portal
🔒 Security Best Practices:
  • Never commit your token to GitHub
  • Use environment variables (we handle this for you!)
  • Reset your token if you think it's compromised
  • Don't share your code publicly with hardcoded tokens
  • Add rate limiting to prevent spam in your bot
⚠️ Common Mistakes to Avoid:
  • ❌ Hardcoding the token in your code (use process.env.DISCORD_TOKEN)
  • ❌ Forgetting to enable Message Content Intent
  • ❌ Using client.login("your_token_here") instead of client.login(process.env.DISCORD_TOKEN)
  • ❌ Not inviting the bot to a server before testing
  • ❌ Copy-pasting code with extra spaces in the token
🚀 Pro Tips:
  • 💾 Save your code locally as a backup
  • 📝 Use comments in your code for clarity
  • 🔄 Test changes on your PC before deploying here
  • 📊 Monitor your bot's performance in Discord
  • 💬 Join Discord.js support server for help: discord.gg/djs
